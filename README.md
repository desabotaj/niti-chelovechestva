# Threads of Humanity

## RU

Интерактивный data-art проект для выпускной квалификационной работы.
500 лет синтетической генеалогии и миграций человечества визуализированы
как светящиеся нити на 3D-глобусе.

### Что это

Сотни миллионов синтетических людей генерируются Spark-конвейером на основе
калиброванной демографической и миграционной модели, управляемой каталогом
из 74 исторических событий (эпидемии, колонизации, войны, постколониальные
потоки и т.д.). Миграции агрегируются в H3-гексагоны и top-дуги по декадам,
после чего визуализируются во фронтенде React + deck.gl как кинематографичный
интерактивный глобус.

Это не буквальная реконструкция истории, а *правдоподобная синтетическая модель*
«реки человечества».

### Структура репозитория

```
.
├── backend/                # Big-data пайплайн (Python + PySpark + DuckDB/H3)
│   ├── generator/          # Демография / миграции + исторические события
│   ├── pipeline/           # Генерация в Spark, агрегация DuckDB, экспорт
│   └── data/               # Сгенерированные parquet (gitignored)
├── frontend/               # Интерактивная визуализация (Vite + React + TS + deck.gl)
│   ├── src/
│   │   ├── components/     # Globe, TimeSlider, LineageExplorer, IntroOverlay...
│   │   ├── layers/         # H3-плотность, дуги миграций, родовые нити
│   │   └── store/          # Zustand stores (время, выбор, слои)
│   └── public/data/        # Агрегированные тайлы для фронтенда
└── docs/                   # Методология, модель данных и т.д.
```

### Быстрый старт

#### Backend pipeline

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate          # Windows
pip install -r requirements.txt

# Генерация синтетического датасета (tiny / small / medium / big)
python -m pipeline.spark_generate --scale small

# Агрегация в H3-гексагоны + top-дуги миграций по декадам
python -m pipeline.aggregate

# Экспорт в frontend public/data/
python -m pipeline.export
```

По умолчанию `small` дает ~1M людей примерно за минуту на ноутбуке.
`big` нацелен на 300M+ людей, записывает партиционированный по декадам
Parquet (~25 ГБ), рассчитан на кластер, но может запускаться и локально
при достаточных ресурсах.

#### Frontend

```bash
cd frontend
npm install
npm run dev
```

Открыть: http://localhost:5173.

### Примечания по дизайну

- **Почему PySpark + Parquet**: масштабирование от ноутбука до кластера без переписывания; партиции по декадам соответствуют основному паттерну запросов.
- **Почему H3-агрегация**: `H3HexagonLayer` в deck.gl GPU-дружественен и корректно работает на `GlobeView`.
- **Почему deck.gl, а не только Three.js**: батч-рендер слоев держит 100k+ дуг и 10k+ гексов при 60 FPS, при этом можно добавлять кастомный GLSL для свечения «нитей».
- **Почему синтетическая модель, а не Census/UN**: полное покрытие 500 лет, единая гранулярность и воспроизводимость от одного seed.

Подробности: `docs/`.

## EN

Interactive data-art project for a graduation thesis.
Five centuries of synthetic human genealogy and migration are rendered
as glowing threads on a 3D globe.

### What it is

Hundreds of millions of synthetic individuals are generated by a Spark pipeline
using a calibrated demographic and migration model driven by a catalog
of 74 historical events (plagues, colonizations, wars, post-colonial flows, etc.).
Migrations are aggregated into H3 hex tiles and top arcs per decade, then
visualized in a React + deck.gl frontend as a cinematic interactive globe.

This is not a literal reconstruction of history, but a *plausible synthetic model*
of the river of humanity.

### Repository layout

```
.
├── backend/                # Big-data pipeline (Python + PySpark + DuckDB/H3)
│   ├── generator/          # Demographic / migration model + historical events
│   ├── pipeline/           # Spark generation, DuckDB aggregation, export
│   └── data/               # Generated parquet (gitignored)
├── frontend/               # Interactive visualization (Vite + React + TS + deck.gl)
│   ├── src/
│   │   ├── components/     # Globe, TimeSlider, LineageExplorer, IntroOverlay...
│   │   ├── layers/         # H3 density, migration arcs, lineage threads
│   │   └── store/          # Zustand stores (time, selection, layers)
│   └── public/data/        # Aggregated tiles consumed by frontend
└── docs/                   # Thesis chapters: methodology, data model, etc.
```

### Quick start

#### Backend pipeline

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate          # Windows
pip install -r requirements.txt

# Generate synthetic dataset (tiny / small / medium / big)
python -m pipeline.spark_generate --scale small

# Aggregate into H3 hex tiles + top migration arcs per decade
python -m pipeline.aggregate

# Export to frontend public/data/
python -m pipeline.export
```

By default, `small` produces about 1M individuals in roughly one minute
on a laptop. `big` targets 300M+ individuals and writes decade-partitioned
Parquet (~25 GB), intended for cluster runs but still possible on a single
machine with enough RAM and disk.

#### Frontend

```bash
cd frontend
npm install
npm run dev
```

Open: http://localhost:5173.

### Design notes

- **Why PySpark + Parquet**: scales from laptop to cluster without rewrite; decade partitioning matches the core query pattern.
- **Why H3 aggregation**: deck.gl `H3HexagonLayer` is GPU-friendly and works reliably on `GlobeView`.
- **Why deck.gl over plain Three.js**: GPU-batched layers handle 100k+ arcs and 10k+ hex cells at 60 FPS, while still allowing custom GLSL for the signature thread glow.
- **Why synthetic model instead of Census/UN data**: complete 500-year coverage, consistent granularity, and reproducibility from a single seed.

See `docs/` for full methodology.
