# Threads of Humanity

Interactive data art for graduation thesis. 500 years of synthetic human genealogy and migrations rendered as flowing threads on a 3D globe.

## What it is

Hundreds of millions of synthetic individuals are generated by a Spark pipeline using a calibrated demographic + migration model driven by a catalog of 74 historical events (Columbian Exchange, plagues, colonizations, world wars, post-colonial flows, etc). Migrations are aggregated into H3 hex tiles and top-arcs per decade, then visualized in a React + deck.gl front-end as a cinematic interactive globe.

Not a literal reconstruction of history — a *plausible synthetic model* of the river of humanity.

## Quick start

### Backend pipeline

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate          # Windows
pip install -r requirements.txt

# Generate synthetic dataset (configurable scale: tiny / small / medium / big)
python -m pipeline.spark_generate --scale small

# Aggregate into H3 hex tiles + top migration arcs per decade
python -m pipeline.aggregate

# Export to frontend public/data/
python -m pipeline.export
```

Default `small` scale produces ~1M individuals in ~1 minute on a laptop. `big` targets 300M+ individuals — write Parquet, partitioned by decade, ~25 GB on disk; meant for cluster runs but works on a single machine with enough RAM/disk.

### Frontend

```bash
cd frontend
npm install
npm run dev
```

Open http://localhost:5173.

## Design notes

- **Why PySpark + Parquet**: scales from laptop to cluster without rewrite; partitioning by decade matches our query pattern.
- **Why H3 hex aggregation**: deck.gl `H3HexagonLayer` is GPU-friendly and works on `GlobeView` (heatmap/contour layers don't).
- **Why deck.gl over plain Three.js**: GPU-batched layers handle 100k+ arcs / 10k+ hex cells at 60 FPS without hand-rolled shaders for every primitive — and we still drop into raw GLSL for the bespoke "threads" glow.
- **Why a synthetic model rather than real Census/UN data**: full coverage of 500 years, full granularity, and reproducibility from a single seed — the entire dataset is regenerable.

See `docs/` for the full methodology.
